Lab 03

Evolución del sistema académico (Opción A): Programas + Asignaturas + Grupos + Matrícula (reglas de negocio, paginación, seeder, Swagger)

Partiendo del proyecto demo-academico (Lab 02 implementado con Opción A), vas a evolucionar el dominio para construir el primer flujo real de negocio: crear Programas, crear Asignaturas, abrir Grupos y matricular Estudiantes con reglas mínimas.

Punto de partida: ya existe el módulo Estudiantes con: DTOs, ApiResponse, ResponseBuilder, GlobalExceptionHandler, paginación y Seeder con Faker. Verifica Swagger en /swagger-ui.html.
Importante: Este laboratorio se desarrolla solo con Opción A. Mantén el mismo estilo: DTO + Handler + ApiResponse. No mezcles con Opción B.

1) Resultados esperados

  • CRUD funcional de Programa, Asignatura y Grupo.
  • Flujo funcional de Matrícula (POST matricular) con reglas mínimas.
  • Paginación funcional en mínimo 2 listados (ej: Programas y Grupos).
  • Errores controlados por GlobalExceptionHandler devolviendo ApiResponse.
  • Seeder extendido con Faker para Programas/Asignaturas/Grupos (opcional: estudiantes si ya existe).
  • Pruebas realizadas desde Swagger (/swagger-ui.html).

2) Modelo de dominio mínimo (relaciones)

Vas a construir un dominio mínimo con estas entidades y relaciones:

  • Programa (1) — (N) Asignatura
  • Asignatura (1) — (N) Grupo
  • Estudiante (N) — (N) Grupo mediante Matricula
Regla práctica: En JPA se implementa el N..N mediante la entidad intermedia Matricula (no usar @ManyToMany directo en este laboratorio).

3) Configuración requerida (application.yml)

Archivo: src/main/resources/application.yml. Extiende el seeder para múltiples entidades.

    cantidad-grupos: 30
Tip: Si no quieres sembrar estudiantes (porque ya los tienes), puedes dejar cantidad-estudiantes en 0.

package co.edu.demoacademico.model;

import jakarta.persistence.*;

MatriculaServiceImpl (implementación con reglas)

// src/main/java/co/edu/demoacademico/service/MatriculaServiceImpl.java
package co.edu.demoacademico.service;

import co.edu.demoacademico.exception.BusinessException;
import co.edu.demoacademico.exception.NotFoundException;
import co.edu.demoacademico.model.Estudiante;
import co.edu.demoacademico.model.Grupo;
import co.edu.demoacademico.model.Matricula;
import co.edu.demoacademico.repository.EstudianteRepository;
import co.edu.demoacademico.repository.GrupoRepository;
import co.edu.demoacademico.repository.MatriculaRepository;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
@Transactional
public class MatriculaServiceImpl implements MatriculaService {

    private final EstudianteRepository estudianteRepo;
    private final GrupoRepository grupoRepo;
    private final MatriculaRepository matriculaRepo;

    public MatriculaServiceImpl(
            EstudianteRepository estudianteRepo,
            GrupoRepository grupoRepo,
            MatriculaRepository matriculaRepo
    ) {
        this.estudianteRepo = estudianteRepo;
        this.grupoRepo = grupoRepo;
        this.matriculaRepo = matriculaRepo;
    }

    @Override
    public Matricula matricular(Long estudianteId, Long grupoId) {

        Estudiante estudiante = estudianteRepo.findById(estudianteId)
                .orElseThrow(() -> new NotFoundException("Estudiante no encontrado: " + estudianteId));

        Grupo grupo = grupoRepo.findById(grupoId)
                .orElseThrow(() -> new NotFoundException("Grupo no encontrado: " + grupoId));

        // Regla 1: no duplicado
        if (matriculaRepo.existsByEstudianteIdAndGrupoId(estudianteId, grupoId)) {
            throw new BusinessException("El estudiante ya está matriculado en ese grupo");
        }

        // Regla 2: cupo
        long matriculados = matriculaRepo.countByGrupoId(grupoId);
        if (matriculados >= grupo.getCupoMax()) {
            throw new BusinessException("El grupo no tiene cupo disponible");
        }

        Matricula m = new Matricula();
        m.setEstudiante(estudiante);
        m.setGrupo(grupo);
        return matriculaRepo.save(m);
    }
}

package co.edu.demoacademico.controller;

import co.edu.demoacademico.api.ApiResponse;
import co.edu.demoacademico.api.ResponseBuilder;
import co.edu.demoacademico.dto.matricula.MatriculaCreateDTO;
import co.edu.demoacademico.handler.MatriculaHandler;
import co.edu.demoacademico.model.Matricula;
import jakarta.validation.Valid;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/matriculas")
public class MatriculaController {

    private final MatriculaHandler handler;

    public MatriculaController(MatriculaHandler handler) {
        this.handler = handler;
    }

    @PostMapping
    public ResponseEntity<ApiResponse<Matricula>> matricular(@Valid @RequestBody MatriculaCreateDTO in) {
        // Puedes devolver MatriculaDTO si prefieres no exponer entity.
        return ResponseBuilder.created("Matrícula creada", handler.crear(in));
    }
}

6) Reglas de negocio obligatorias

  1. Matrícula duplicada: no permitir matricular el mismo estudiante en el mismo grupo dos veces.
  2. Cupo: no permitir matrícula si matriculados >= cupoMax.
  3. Existencia: si estudiante o grupo no existe → lanzar NotFoundException.
Plus: Puedes agregar en Programa la regla codigo único usando existsByCodigo y una BusinessException.
